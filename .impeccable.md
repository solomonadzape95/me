# solenoid.me — Design Context

## Design Context

### Users

Three audiences sharing one surface:

- **Recruiters and hiring managers** — landing here from LinkedIn or a referral. 20–40 seconds to decide whether to read further. Need: an immediate, credible read on craft and seriousness. They don't care about the framework — they care that the page itself feels considered.
- **Fellow engineers and designers** — friends, peers, future collaborators. They'll inspect the source, scrub the network tab, and notice the typography choices. The site has to survive their scrutiny.
- **Founders / future teammates** — evaluating whether Solomon could be a founding engineer hire. Looking for evidence of taste and finish, not for a feature list.

The job to be done is *proof*. Not a pitch, not a story — proof that this person ships work with real frontend craft. Every element either earns its place by demonstrating that, or it gets cut.

### Brand Personality

**Quiet, considered, slightly weird.** The page does not speak in the first-person marketing voice. It does not say "I am passionate about creating beautiful and useful things." It uses typography, rhythm, and one signature moment to communicate the same thing without ever uttering it.

The owner is a student and founding engineer who works across frontend, web3, and AI tooling, and who cares about motion and typography. The site should feel like the personal letterhead of someone who would notice that you set the dates in tabular figures.

Emotional goal: a 30-second visitor leaves with the feeling *this person is the real thing.* No effort is spent winning them over. The site is allowed to be a little strange — that's the differentiator.

### Aesthetic Direction

**Typographic specimen sheet, not landing page.** Treat the site as a museum-catalog card or a typographer's specimen for a person. Not a marketing surface.

- **Theme**: dual-mode. Light is the default; dark mirrors it.
  - Light: warm paper `oklch(0.97 0.012 75)` on espresso ink `oklch(0.20 0.018 60)`. The neutrals are tinted toward the same warm hue family as the accent so the surfaces and text feel cohesive.
  - Dark: warm ink `oklch(0.13 0.014 60)` background on a cream-grey `oklch(0.92 0.008 75)` foreground. Slightly warmer than the typical cool-dark default.
- **Hot accent**: a single vermillion `oklch(0.62 0.20 32)` in light mode, shifted to a slightly hotter cinnabar `oklch(0.68 0.22 30)` in dark mode for perceptual brightness. Accent appears in exactly three places: the trailing display-name punctuation, the active nav indicator, and the hover state of the one signature interaction. Nowhere else.
- **Composition**: asymmetric. Wide screens earn a sticky left rail that holds page-level metadata (name, role, located, timezone, contact) and section labels that *hang off the left* rather than capping their content (Tufte sidenote style). Narrow screens collapse to a single column with the labels promoted to inline pre-headers. Hairline `1px` rules carry structure. **No `rounded-2xl` card spam. No background-fill cards. No drop shadows.**
- **Information display**: every page reads like an archival entry, not a paragraph wall. Projects render as **specimen entries**: serif display title, mono label strip with `tech · year · status · links`, one paragraph blurb, one hero shot inline. About reads like a CV transcript with monospace dates flush against prose. Skills render as a **typographic matrix** — a 5-column table with monospace category labels and grotesque chip values, not an icon grid.

**Anti-references** (do not look like these):
- The current solenoid.me — sky/magnets/shutters/3D-route decoration. That era is over.
- Generic Vercel-template dev portfolio: Geist or Inter, zinc cards, lucide icons above headings, soft gradient hero, "currently building →" repeated three times.
- Cyberpunk web3: cyan-on-black, purple-to-blue gradients, glowing borders, holographic textures.
- shadcn/ui card spam: rounded rectangles, drop shadows, identical card grids of three.
- "Big landing page" minimalism: 120px hero headline in a soft sans, gradient mesh blur, three feature columns.

**References** (the lineage):
- Typography specimens: Production Type, Klim Type Foundry, Pangram Pangram specimen pages.
- Tufte handouts and edwardtufte.github.io/tufte-css for the sidenote rail.
- Late-19th-century museum catalog cards: tabular metadata, hand-set sub-labels, generous margins.
- Tom Sachs studio output: monospace labels on warm-paper, hand-stamped urgency, *workman's certificate* energy.

### Type System

Two families, both free via `next/font`. The reflex picks (Fraunces, Newsreader, Instrument Serif, Inter, Space Grotesk, IBM Plex anything, DM Serif, Geist) are off the table — they are the 2024–2025 AI-portfolio monoculture. The system commits to the binary serif × monospace split — body prose is set in mono on purpose, leaning the site toward dossier / system-readout energy.

- **Display — Spectral** (Production Type, variable, weights 200–800, italic). A high-contrast Garamond-influenced serif designed for on-screen reading. Used only at display sizes for the name, page titles, and (occasionally) project names in selected lists. Spectral has enough quirk in its italic and its terminal flourishes to feel personal without veering into novelty.
- **Everything else — DM Mono** (Colophon Foundry for Google Fonts, weights 300/400/500, italic). The DM family is interesting because only DM Sans / DM Serif Display / DM Serif Text are 2024-2025 reflex picks — DM Mono is comparatively underused. Carries body prose, labels, metadata, dates, links, code, status, and any other text that isn't a display-size heading. Tabular figures via `font-variant-numeric: tabular-nums`.

**Role separation is binary.** Spectral is *only* used at display sizes for headings — never for body text. DM Mono carries everything else, including paragraphs. Crossing this rule is the fastest way to destroy the system.

Type scale (fluid for marketing/content pages, fixed for any in-app surfaces):
- Display XL (name on `/`): `clamp(3.5rem, 8vw, 6.5rem)` Spectral 400 italic-optional
- Display L (page titles): `clamp(2.25rem, 5vw, 3.5rem)` Spectral 400
- H2 (section headings, hung in rail): `1.125rem` Spectral 500
- Body: `1rem` (16px) Schibsted Grotesk 400, line-height `1.6`
- Small body / captions: `0.875rem` Schibsted Grotesk 400
- Mono label: `0.75rem` Fragment Mono, `letter-spacing: 0.05em`, uppercase optional
- Mono inline: `0.875rem` Fragment Mono

OpenType features turned on in `globals.css`: `kern`, `liga`, `clig`, `calt`, plus `tnum` and `lnum` on mono.

### Layout & Density

- 4pt spacing scale, semantic tokens: `--space-xs 4px`, `--space-sm 8px`, `--space-md 12px`, `--space-lg 16px`, `--space-xl 24px`, `--space-2xl 40px`, `--space-3xl 64px`, `--space-4xl 96px`.
- **Wide layout (≥1024px)**: two-column grid. Left rail is `220–280px`, sticky from top, holds the global metadata block and the currently-active section label hanging off the left. Right column is the content stream, capped at `65ch` for prose, `60ch` for paragraphs, full bleed for hero shots.
- **Narrow layout (<1024px)**: single column. The left-rail metadata becomes an inline header block. Section labels go from "hanging in the rail" to "small monospace pre-header above the section."
- **Section anatomy**: a thin `1px` hairline rule at the bottom marks each section boundary. The hairline is `--color-rule` (a low-chroma ink at ~12% opacity). No box backgrounds, no card framing.
- **Project entries** are NOT cards. They are typographic specimens: serif title, mono label strip on the line below, single paragraph, single hero image, with `48–64px` between entries.

### Memorable Moment

**Two complementary moments**, one passive and one interactive:

1. **Display-name hue cascade on first paint.** On `/`, the name "Solomon Adzape" loads with each character initially set to `--color-rule` (the same near-invisible neutral as the hairlines). A staggered cascade transitions each character to `--color-ink` over `360ms` total, with `28ms` per character, using `cubic-bezier(0.16, 1, 0.3, 1)`. After the cascade, no other motion happens unless the user initiates it.

2. **Site graph (right rail).** At `xl` viewports (≥1280px), a small SVG node-graph sits in a third column. Nodes for home / about / projects / contact / resume; on a project page, the slug becomes a sixth node attached to projects. Edges are the explicit content-link graph (not the rail nav, which is fully connected and uninteresting to draw). The current node is filled in `--color-accent` and slightly larger; incident edges shade to `--color-accent-soft`. Click navigates. This is the frontend-craft proof for visitors who'd otherwise have nothing visual to chew on after the cascade settles.

Everywhere else, hover states are `80–120ms` linear-ish easings on opacity and underline-offset only. No route transitions, no scroll-triggered reveals, no page-load shimmer.

### Design Principles

1. **The site is the proof, not the pitch.** No first-person marketing voice ("I am passionate about..."), no value-prop sentences, no call-to-action buttons. Demonstrate craft through typography, rhythm, and one signature moment. The visitor concludes what they need to conclude on their own.
2. **Three faces, three roles, no overlap.** Spectral carries the name. Schibsted Grotesk carries the prose. Fragment Mono carries the data. Crossing roles is the fastest path back to AI slop.
3. **Hairlines, not cards.** Structure comes from `1px` rules, generous space, and hanging side labels — never from background-filled boxes, rounded corners, or drop shadows. If a section "needs a card," it needs a better layout instead.
4. **Asymmetry is the signal.** Wide screens earn a left rail and hanging labels; narrow screens collapse to a single column. The composition itself must adapt, not just shrink-to-fit.
5. **Vermillion is rare and earned.** The accent color appears in four places sitewide: the active-nav item (text + underline, no arrow), the current node of the site graph, hover affordances on interactive surfaces, and the focus ring. Decorative use is forbidden — including arrows, dots, or stripes purely as visual chrome.

## Notes for implementation

- Tokens live in `app/globals.css` under a `@theme` block (Tailwind v4 CSS-first config). The dark theme is keyed off `[data-theme='dark']` on `<html>`; default is `prefers-color-scheme`, overridable via a small client toggle that writes to `localStorage`. A blocking inline script in `<head>` applies the theme before paint to prevent FOUC.
- `next/font` loads Spectral and DM Mono as CSS variables: `--font-spectral` and `--font-dm-mono`. They are aliased in `@theme` to `--font-display` / `--font-body` / `--font-mono`. Use via Tailwind utilities (`font-display`, `font-body`, `font-mono`).
- The display-name cascade is CSS-only: each character is wrapped in a `<span>` with an inline `--i` custom property holding its index, and an `animation-delay: calc(var(--i) * 28ms)`. There is no `cascade-tail` element — the name renders alone.
- The site graph (`components/site-graph.tsx`) is a static SVG with hard-coded node positions; the only runtime input is `usePathname()`. No d3, no canvas, no force simulator. Navigation uses `useRouter().push` so SVG elements can act as nav targets without nesting `<a>` inside an SVG `<g>`.
- The avatar (`components/avatar.tsx`) is a DiceBear `adventurer-neutral` SVG fetched once and stored in `public/avatar.svg` plus `app/icon.svg` (Next auto-routes the latter as the favicon at every size). The in-page version sits next to the display name in the rail.
- View Transitions API is used for in-app navigation (Next 15 supports it). Transitions are a `120ms` linear crossfade only — no shared-element acrobatics. With `prefers-reduced-motion`, transitions are disabled.
- Accessibility minimum: focus rings use the accent at low chroma (`oklch(0.62 0.20 32 / 0.5)`) with a `2px` outline and `2px` offset; never `outline: none`. Color contrast ≥4.5:1 for body, ≥3:1 for the active-nav vermillion against either background.
- Bans, enforced in review: `border-left`/`border-right` greater than 1px as a colored stripe; `background-clip: text` with a gradient; gradient text of any form; `rounded-2xl`; drop shadows on cards; lucide icons above headings; full-bleed gradient hero backgrounds; decorative arrows (`→`, `←`, `·`) as nav or list-item indicators.
